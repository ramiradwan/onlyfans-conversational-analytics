# Design sync notes

- Source shape is `package`, but this repository is an application rather than a published component library. The converter intentionally uses the committed curated source entry `.design-sync/entry.ts`; `componentSrcMap` defines the public visual surface while the entry also exposes required preview providers and deterministic fixture hooks.
- `npm run build` outputs the application into `../app/static/dist`; it does not create a reusable package entry or declaration tree. The sync therefore relies on source-level contracts. Generated `.d.ts` files currently use permissive index signatures, so future syncs should add and maintain explicit `dtsPropsFor` contracts for components whose prop APIs must be strongly constrained.
- The current component surface has two source exports named `MessageBubble`. The sync exposes the newer inbox component (`src/components/inbox/MessageBubble.tsx`) and omits the older legacy bubble (`src/components/MessageBubble.tsx`) because a single bundle namespace cannot faithfully carry both under the same name.
- Theme styling is MUI v7 CSS-in-JS. `.design-sync/styles.css` carries the app's global body/color-scheme rules without its package-relative font import; `src/assets/fonts/fonts.css` and local WOFF2 files are explicitly shipped through `extraFonts`.
- The converter's simple path resolver does not probe directory indexes. `.design-sync/tsconfig.json` mirrors the app aliases and adds explicit `index.ts` / `index.tsx` targets for directory imports; keep it aligned with the app's alias map.
- Full rich-preview authoring was selected for the curated visual surface on 2026-07-19.
- The 2026-07-19 Modern Analytics translation keeps the remote `templates/modern-analytics/**` showcase as reference-only design data. Production and previews use the repository theme, real routes, role gates, and canonical Bridge snapshots; the sync must never overwrite or upload the remote template folder.
- `CreatorDashboardView` and `OperatorInboxView` accept optional read-only store seams whose `getState()` contracts are deliberately narrowed to the exact Bridge state slices each view reads. `createPreviewBridgeStore()` owns a fixed live snapshot (including connected session/Agent/System state) so dashboard and shell previews are deterministic and make no backend or WebSocket calls.
- Shell contracts now expose optional `AppAppBar.headerHeight` and `AppDrawer.mobileDrawerWidth` / `onDrawerClose`; `onDrawerToggle` remains an optional compatibility fallback on `AppDrawer`.
- The production `App` owns its `ThemeProvider`, `BrowserRouter`, and live WebSocket lifecycle, while the design-agent runtime expects externally wrapped, deterministic compositions. It is intentionally excluded from `componentSrcMap`; use `AppShell` plus individual views for prototypes.
- The shipped Inter files cover weights 400, 600, and 700. Theme aliases `medium` to 600 and `extrabold` to 700 so previews and generated designs never rely on synthetic 500/800 faces.

## Re-sync risks

- Component inclusion is explicit in `componentSrcMap`; newly added visual exports will not appear until that sparse map is extended deliberately.
- Source-level synthesized bundles have weaker automatic `.d.ts` extraction than a published package. Re-check generated contracts after component prop changes and keep `dtsPropsFor` overrides current. The output format emits a non-generic `<Name>Props` interface, so `AsyncContent<T>` is deliberately represented with `any[]` to preserve callback usability; `Panel.sx` structurally approximates the serializable object/callback/array subset of MUI `SxProps` because the uploaded contract cannot depend on MUI package types.
- Application views and layout components may depend on Zustand stores, router context, or browser APIs. Their authored previews must own deterministic fixtures/context and must not make live backend or WebSocket calls.
- Preview data must remain static and representative; keep `createPreviewBridgeStore()` timestamps, UUIDs, counts, and message fixtures fixed. Update its canonical snapshot deliberately when protocol contracts change rather than seeding production singletons during render.
- The modern shell preview initializes the bundled singleton transport store once through the idempotent `seedPreviewShellStore()` helper to make its live-status chip deterministic without publishing during component render. The dashboard and Inbox previews use isolated stores instead.
- The dashboard preview is intentionally canonical-only: revenue, spend, subscribers, conversion, time-period deltas, and generated query answers from the reference template are unsupported and must not enter preview fixtures.
- Authored cards cover representative stable states rather than every transport transition. `CreatorDashboardView` is visually graded on its canonical live desktop snapshot and `AppAppBar` on live chrome; loading, resyncing, degraded, error, and unavailable behavior is regression-tested in product tests and should gain additional cards only when isolated per-story store seams can keep multi-cell previews independent.
- The bundle assumes the repository's pinned npm lockfile and Node `>=18`; Playwright browser revision compatibility must be checked against the installed `.ds-sync` Playwright package before validation.
- No network-fetched fonts or styling assets are accepted; Inter is sourced from the repository's local WOFF2 files.
- The remote DesignSync anchor must be freshly fetched before an upload. The exported ZIP's `_ds_sync.json` is only a local fallback for diff/validation when the authenticated `claude_design` MCP is unavailable, never evidence that the remote anchor is current.
